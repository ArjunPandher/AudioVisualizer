Arjun's Audio Visualiser Documentation

Introduction

This is an audio visualiser library, making it easy for developers to quickly add audio-reactive components to their website.
Add <script src="bav.js"></script> to your HTML <head> tag.

AudioVisGenerator

This can be used to generate all objects in the library, as well as animate them.

Usage

Create a new AudioVisGenerator using const audioVisGen = new AudioVisGenerator(audioPath, audioElementId) Where audioPath is a url to an mp3 file, and audioElementId is the ID of the audio player you are using. Make sure to use AudioVisGenerator.setUp on some form of user input before calling any of its animate methods. This is mandatory for most modern browsers. You can instantiate multiple AudioVisGenerators for different audio sources and different audio players.
List of methods
Name Parameters Description
AudioVisGenerator.setUp() Sets up audio processing modules that aren't allowed to be set up upon window creation. Must be called after some user interation with the webpage.
AudioVisGenerator.addCanvas(canvas, canvasType, primaryColor, secondaryColor) canvas: Canvas
canvasType: Integer
primaryColor(optional): RGB/Hex Color
secondaryColor(optional): RGB/Hex Color 		Creates and returns a new AnimCanvas object. This is the recommended way to add a new canvas to your program. canvasType must be in range 0-1, where each integer corresponds to a canvas animation type as follows: {0: "bar", 1: "circle"}
AudioVisGenerator.addElement(element, animationType, animationParams) element: HTML Element
animationType: Integer
animationParams: Array[Object] 		Creates and returns a new AnimElement object. This is the recommended way to add a new animated element to your program. animationType must me in range 0-8, where each integer corresponds to an animation type as follows: {0: "vertical", 1: "horizontal", 2: "rotation", 3: "scaleX", 4: "scaleY", 5: "skewX", 6: "skewY", 7: "color", 8: "backgroundColor"}.
AudioVisGenerator.removeCanvas(canvasID) canvasID: Integer Removes a canvas with ID canvasID from the AudioVisualizer.
AudioVisGenerator.removeElement(elementID) elementID: Integer Removes an element with ID elementID from the AudioVisualizer.
AudioVisGenerator.animateAllCanvases() Animates all of the canvases in the AudioVisualizer.
AudioVisGenerator.animateAllElements() Animates all of the elements in the AudioVisualizer.
AudioVisGenerator.animateCanvas(animCanvas) animCanvas: AnimCanvas Animates a specific AnimCanvas.
AudioVisGenerator.animateElement(animElement) animElement: AnimElement Animates a specific AnimElement.

AnimCanvas

The AnimCanvas adds a bunch of functionality around HTML canvases, automatically adjusting it to sync to the audio of the AudioVisGenerator it was created in. It is highly reccomended to only create new AnimCanvases through the AudioVisGenerator.addCanvas() method.

Usage

Create a new AnimCanvas using const animCanvas = AudioVisGenerator.addCanvas(canvas, canvasType, primaryColor, secondaryColor) See the documentation for AudioVisGenerator.addCanvas() above.
List of methods
Name Parameters Description
AnimCanvas.setCanvasType(canvasType) canvasType: String TO BE CHANGED ONCE IM DONE PACKAGING
AnimCanvas.removeCanvas() Removes the AnimCanvas element from the DOM. Generally shouldn't be used on its own outside of AudioVisGenerator.removeCanvas() for optimization purposes.
AnimCanvas.hideCanvas() Hides the AnimCanvis from the DOM.
AnimCanvas.showCanvas() Reveals the AnimCanvis in the DOM, if AnimCanvas.hideCanvas() was called prior on this AnimCanvas.
AnimCanvas.makeDraggable() Allows the canvas to be dragged across the screen as the user wishes. Canvas must have the position: absolute CSS attribute.
AnimCanvas.makeUndraggable() Prevents the canvas from being draggable, if AnimCanvas.makeDraggable() was called prior on this AnimCanvas.
AnimCanvas.setDefaultPosition(position) position: String Changes the default position CSS attribute that the AnimCanvas uses. Only to be used if one plans on using makeDraggable AND wants to change the canvas's position attribute after creation.

AnimElement

The AnimElement adds a bunch of functionality to any HTML element you wish, automatically adjusting it to sync to the audio of the AudioVisGenerator it was created in. It is highly reccomended to only create new AnimElements through the AudioVisGenerator.addElement() method.

Usage

Create a new AnimElement using const animElement = AudioVisGenerator.addElement(element, animationType, animationParams) See the documentation for AudioVisGenerator.addElement() above.
List of methods
Name Parameters Description
AnimElement.setAnimationType(animationType, animationParams) animationType: String
animationParams: Array[Object] 		TO BE CHANGED ONCE IM DONE PACKAGING
AnimElement.removeElement() Removes the AnimElement element from the DOM. Generally shouldn't be used on its own outside of AudioVisGenerator.removeElement() for optimization purposes.
AnimElement.hideElement() Hides the AnimElement from the DOM.
AnimElement.showElement() Reveals the AnimElement in the DOM, if AnimElement.hideElement() was called prior on this AnimElement.
AnimElement.makeDraggable() Allows the element to be dragged across the screen as the user wishes. Element must have the position: absolute CSS attribute.
AnimElement.makeUndraggable() Prevents the element from being draggable, if AnimElement.makeDraggable() was called prior on this AnimElement.
AnimElement.setDefaultPosition(position) position: String Changes the default position CSS attribute that the AnimElement uses. Only to be used if one plans on using makeDraggable AND wants to change the element's position attribute after creation.

Animation Types

The AnimElement has a wide variety of animation options, but each one requires different arguments to properly function. The following table documents all of the different types of animations available in this library, and the arguments they require.
List of animation types
Name Parameters Description
0: "verticle" Array[0](maxMoveAmount): Integer maxMoveAmount is the maximum verticle distance in pixels you would like the element to move.
1: "horizontal" Array[0](maxMoveAmount): Integer maxMoveAmount is the maximum horizontal distance in pixels you would like the element to move.
2: "rotation" Array[0](maxRotateAmount): Integer maxRotateAmount is the maximum rotation amount in degrees you would like the element to rotate.
3: "scaleX" Array[0](maxScaleAmount): Integer maxScaleAmount is the maximum horizontal ratio you would like the element to scale.
4: "scaleY" Array[0](maxScaleAmount): Integer maxScaleAmount is the maximum verticle ratio you would like the element to scale.
5: "skewX" Array[0](maxSkewAmount): Integer maxSkewAmount is the maximum horizontal ratio you would like the element to skew.
6: "skewY" Array[0](maxSkewAmount): Integer maxSkewAmount is the maximum verticle ratio you would like the element to skew.
7: "color" Array[0](color1): {red: Integer, blue: Integer, green: Integer} Array[1](color2): {red: Integer, blue: Integer, green: Integer} color1 and color2 are two objects, each with three integers corresponding to the RGB values you would like the DOM element's color attribute to transition from and to. For best results, choose colors close to one another in RGB values.
8: "backgroundColor" Array[0](color1): {red: Integer, blue: Integer, green: Integer} Array[1](color2): {red: Integer, blue: Integer, green: Integer} color1 and color2 are two objects, each with three integers corresponding to the RGB values you would like the DOM element's backgroung-color attribute to transition from and to. For best results, choose colors close to one another in RGB values.
